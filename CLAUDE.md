## Project Overview

Sharp Cooking Desktop is a [Wails](https://wails.io/) application that wraps the Sharp Cooking PWA (Vue 3 + TypeScript frontend) in a native desktop shell using Go. The frontend lives as a git submodule in `./frontend/`.

## Commands

### Desktop App (Go + Wails)
```bash
wails dev        # Live development mode — starts Vite dev server with hot reload
wails build      # Build redistributable desktop app
```

### Frontend (run from `./frontend/`)
```bash
yarn install                          # Install dependencies
yarn dev                              # Dev server on port 3000
yarn build                            # TypeScript check + Vite build
npx vue-tsc --noEmit                  # TypeScript type check only
npx playwright test {file}            # Run a specific E2E test file
```

## Architecture

### Wails Bridge
- `main.go` — Wails app entry point; embeds `frontend/dist` as static assets
- `app.go` — Go `App` struct bound to the frontend; methods exposed here are callable from JavaScript via `wailsjs/go/`
- `wailsjs/` — Auto-generated JS/TS bindings (regenerated by Wails on `wails dev`/`wails build`)

### Frontend (submodule at `./frontend/`)
See `frontend/AGENTS.md` for the full conventions. Key points:

- **Framework**: Vue 3 Composition API + TypeScript
- **Routing**: Hash-based (`/#/route`), file-based via `vite-plugin-pages` — add `.vue` files to `src/pages/` to create routes
- **State**: Reactive store in `src/services/store.ts` via Vue's provide/inject; **never access recipes directly from state**, use `getRecipes()` / `getRecipesByCategory()` from `dataService`
- **i18n**: i18next; translation files at `public/locales/{en,pt}/translation.json`; key pattern `pages.{pageName}.{keyName}`
- **Navigation**: `state.menuOptions` drives `TopBar.vue`; `RecipeList.vue` overwrites `menuOptions` in `onMounted`, so menu items must be added in **both** `src/pages/index.vue` and `src/components/RecipeList.vue`
- **PDF export**: `src/services/recipeBookExportService.ts` + `src/helpers/pdfHelpers.ts` using jspdf

### E2E Tests
- Playwright; config in `frontend/playwright.config.ts`
- Use `getByTestId()` first, then accessibility selectors
- Always call `await setup(page)` in `beforeEach`
- Never use `page.evaluate()` for ES module imports, never access IndexedDB directly
- Test across chromium, webkit, Mobile Chrome, Mobile Safari
